## 🔍 Advanced Port Scanner - Complete Guide

### Overview
The **Advanced Port Scanner** (`advancedscanner.py`) supports multiple scanning methodologies to detect open, closed, and filtered ports. Each method has different characteristics, speed, and stealth levels.

### Scanner Methods (`-M` flag)

| Method | Flag | Description | Speed | Stealth | Privilege | Best For |
|--------|------|-------------|-------|---------|-----------|----------|
| **TCP Connect** | `-M tcp` | Full 3-way handshake (SYN→SYN-ACK→ACK) | Slow | ❌ Low | None | Accuracy, firewalls |
| **SYN Stealth** | `-M syn` | Half-open connection (SYN→SYN-ACK only) | Fast | ✅ High | Admin/Root | IDS evasion |
| **ICMP Ping** | `-M icmp` | ICMP Echo requests for host discovery | Very Fast | Medium | Admin/Root | Initial recon |
| **UDP Scan** | `-M udp` | Connectionless UDP probes | Slow | Medium | None | Service detection |
| **FIN/NULL/Xmas** | `-M fin` | TCP flags scanning for stealth | Fast | ✅ Very High | Admin/Root | Deep firewall analysis |

### CLI Flags Reference

```bash
# Scanning Parameters
-H, --host          Target hostname or IP address (required)
-p, --ports         Specific ports: 22,80,443 or range: 1-1000
-r, --range         Port range: 1-10000

# Scanning Modes
-x, --preset        Use preset port lists (web, ssh, database, mail, dns, directory, monitoring, ntp, vpn, common, all)
-M, --method        Scan method: tcp, syn, icmp, udp, fin (default: tcp)
-i, --interactive   Interactive mode with menu

# Information & Help
--list              Show all available presets and descriptions
--info PRESET       Show details for specific preset
--help              Show full help message
```

### Preset Port Lists

The scanner includes pre-configured port lists for common services:

- **web**: 80, 443, 3000, 5000, 8080, 8443, 9000, 8888

### Usage Examples

#### 1. TCP Connect Scan (Default - No Privileges Required)
```bash
# Basic TCP scan on common web ports
python advancedscanner.py -H 192.168.1.100 -x web

# TCP scan on specific ports
python advancedscanner.py -H 192.168.1.100 -p 22,80,443,3389

# TCP scan on port range
python advancedscanner.py -H 192.168.1.100 -r 1000-2000
```

#### 2. SYN Stealth Scan (Requires Admin/Root)
```bash
# Fast SYN scan on database preset
python advancedscanner.py -H 192.168.1.100 -x database -M syn

# SYN scan on specific ports
python advancedscanner.py -H 192.168.1.100 -p 22,3306,5432 -M syn
```

#### 3. ICMP Ping Scan (Requires Admin/Root)
```bash
# Initial host discovery with ICMP
python advancedscanner.py -H 192.168.1.100 -x web -M icmp

# Fast network sweep
python advancedscanner.py -H 192.168.1.0 -M icmp
```

#### 4. UDP Scan (No Privileges Required)
```bash
# UDP scan for DNS, SNMP, NTP services
python advancedscanner.py -H 192.168.1.100 -p 53,161,123 -M udp

# UDP on mail preset
python advancedscanner.py -H 192.168.1.100 -x mail -M udp
```

#### 5. FIN/NULL/Xmas Stealth Scan (Requires Admin/Root)
```bash
# Most stealthy scan type
python advancedscanner.py -H 192.168.1.100 -p 22,80,443 -M fin

# Complete stealth scan on web services
python advancedscanner.py -H 192.168.1.100 -x web -M fin
```

#### Interactive Mode
```bash
# Launch interactive menu with guided scanning
python advancedscanner.py -i

# Show all available presets with details
python advancedscanner.py --list

# Get information about a specific preset
python advancedscanner.py --info database
```

### Port States Explained

The scanner reports three main port states:

- **[OPEN]** - Port is actively listening and accepting connections
- **[CLOSED]** - Port is accessible but not listening (reset received)
- **[FILTERED]** - Port cannot be reached (firewall is dropping packets)

### Method Behavior & Detection

```
TCP Connect (-M tcp):
  Performs full 3-way TCP handshake
  ✓ Works without elevated privileges
  ✓ High accuracy
  ✗ Slow and easily detectable
  ✗ Logged by the target system

SYN Stealth (-M syn):
  Sends SYN, receives SYN-ACK, then RST (no full handshake)
  ✓ Faster than TCP Connect
  ✓ Less logging on older systems
  ✗ Requires admin/root privileges
  ✗ Modern firewalls detect it
  
ICMP Ping (-M icmp):
  Uses ICMP Echo Requests
  ✓ Very fast host discovery
  ✓ Good for initial recon
  ✗ Requires admin/root privileges
  ✗ Many firewalls block ICMP
  
UDP Scan (-M udp):
  Sends empty UDP packets
  ✓ Detects UDP services
  ✓ Works without privileges
  ✗ Slow (ICMP rate limiting)
  ✗ Open|Filtered states hard to distinguish
  
FIN/NULL/Xmas (-M fin):
  Sends TCP packets with unusual flag combinations
  ✓ Most stealthy (bypasses many simple firewalls)
  ✓ Doesn't trigger alarms on old systems
  ✗ Requires admin/root privileges
  ✗ RFC 793 compliance varies by OS
  ✗ Unreliable on modern systems
```

### Privilege Requirements

| Method | Windows | Linux | macOS |
|--------|---------|-------|-------|
| TCP Connect | ✓ User | ✓ User | ✓ User |
| SYN Stealth | ⚠️ Admin | ⚠️ Root | ⚠️ Root |
| ICMP Ping | ⚠️ Admin | ⚠️ Root | ⚠️ Root |
| UDP Scan | ✓ User | ✓ User | ✓ User |
| FIN/NULL/Xmas | ⚠️ Admin | ⚠️ Root | ⚠️ Root |

**Note:** If you lack privileges for SYN/ICMP/FIN scans, the scanner will gracefully fall back to TCP Connect.

### Tips & Best Practices

1. **Start Simple**: Begin with TCP Connect (`-x web`) to verify target is reachable
2. **Use Presets**: Preset lists are optimized for common service detection
3. **Combine Methods**: Use multiple methods on same target for better coverage
4. **Check Filtering**: ICMP blocked? Try TCP. Still nothing? Target might be offline
5. **Respect Limits**: Large port ranges (-r 1-65535) take hours to scan
6. **Document Results**: Save results for gap analysis and compliance
